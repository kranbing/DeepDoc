# 成员二：图谱存储与查询模块 - 任务完成说明

## 模块概述

本模块负责知识图谱的持久化存储和高级查询，是连接"抽取层"与"展示层"的中间件。采用抽象接口 + 多后端实现的架构，当前提供 JSON 本地存储，预留 Neo4j 图数据库接口。

---

## 完成的任务

### 1. 图谱存储抽象接口 (`graph_store.py`)

定义了 `GraphStore` 抽象基类，规范 11 个标准接口方法：

| 方法 | 签名 | 功能 |
|------|------|------|
| `add_nodes` | `(nodes: List[Dict]) -> int` | 批量添加节点，返回新增数 |
| `add_edges` | `(edges: List[Dict]) -> int` | 批量添加边，返回新增数 |
| `get_node` | `(node_id: str) -> Optional[Dict]` | 按 ID 获取单个节点 |
| `get_neighbors` | `(node_id: str, relation: str = None) -> List[Dict]` | 一跳邻居查询，可按关系过滤 |
| `find_path` | `(source_id: str, target_id: str, max_hops: int = 2) -> List[List[str]]` | BFS 路径查找 |
| `search_by_name` | `(name: str, fuzzy: bool = True) -> List[Dict]` | 名称搜索（精确/模糊） |
| `search_by_type` | `(entity_type: str) -> List[Dict]` | 按类型查询所有节点 |
| `get_all_nodes` | `() -> List[Dict]` | 获取全部节点 |
| `get_all_edges` | `() -> List[Dict]` | 获取全部边 |
| `get_stats` | `() -> Dict` | 图谱统计信息 |
| `save` | `(path: str) -> None` | 持久化到文件 |
| `load` | `(path: str) -> None` | 从文件加载 |

设计原则：
- 所有方法接受/返回 Python 原生 dict，避免引入 ORM 绑定
- 邻居查询返回边属性（`_edge_relation`, `_edge_confidence`, `_edge_evidence`），便于上层展示
- 路径查找支持 `max_hops` 限制，防止大规模图上 BFS 爆炸

### 2. JSON 本地存储实现 (`json_store.py`)

基于 Python dict 的内存图存储，支持 JSON 文件持久化。

#### 核心数据结构

```
_nodes: Dict[str, Dict]          # id → node（主存储）
_edges: List[Dict]               # 边列表（索引化）
_adjacency: Dict[str, List[int]] # node_id → [edge_indices]（邻接表）
_name_index: Dict[str, List[str]] # lowercase_name → [node_ids]（名称索引）
_type_index: Dict[str, List[str]] # type → [node_ids]（类型索引）
```

#### 关键实现细节

**节点添加（`add_nodes`）**：
- 新节点：写入 `_nodes`，同时更新 `_name_index` 和 `_type_index`
- 已存在节点：合并模式 — 仅填充缺失字段，不覆盖已有值
- 返回实际新增数（跳过重复）

**边添加（`add_edges`）**：
- 写入前检查重复：同 (source, target, relation) 的边已存在时，保留置信度更高的
- 邻接表双向索引：`_adjacency[src]` 和 `_adjacency[tgt]` 都记录边索引，支持无向遍历

**名称搜索（`search_by_name`）**：
- 三级搜索策略：
  1. 精确匹配：`_name_index[name_lower]` → O(1) 查找
  2. 模糊匹配：`SequenceMatcher` 相似度 ≥ 0.6 → 遍历名称索引
  3. 子串匹配：`name_lower in indexed_name or indexed_name in name_lower`
- 结果按匹配分数降序排列

**路径查找（`find_path`）**：
- BFS 广度优先搜索，逐层扩展
- `max_hops=2` 限制搜索深度（默认两跳）
- 找到目标立即返回当前层所有路径（不继续搜索更深路径）
- 已访问节点去重，避免环路

**JSON 持久化（`save`/`load`）**：
- `save`：将 `_nodes` 和 `_edges` 序列化为 JSON，附带 metadata（统计信息 + 时间戳）
- `load`：清空所有索引后重新 `add_nodes` + `add_edges`，保证索引一致性
- 编码：`ensure_ascii=False`，支持中文实体名称

### 3. Neo4j 存储接口 (`neo4j_store.py`)

可选的图数据库后端，使用 Cypher 查询语言。

#### 实现特点

- **可选依赖**：`try: from neo4j import GraphDatabase` — 未安装时 `HAS_NEO4J=False`，不影响其他模块
- **MERGE 语义**：节点使用 `MERGE` 避免重复，边使用 `MERGE` + `SET` 更新属性
- **动态标签**：节点类型直接作为 Neo4j 标签（如 `:Method`, `:Dataset`）
- **属性透传**：节点/边的所有非关键字段自动映射为 Neo4j 属性
- **路径查询**：使用 Neo4j 的 `[*1..max_hops]` 变长路径语法
- **导出兼容**：`save()` 方法从 Neo4j 读取全量数据后写入 JSON，与 JsonGraphStore 格式一致

#### 全部 11 个接口方法均已实现

每个方法都通过 `self._driver.session()` 执行 Cypher 查询，返回与 `JsonGraphStore` 相同格式的结果。

### 4. 查询引擎 (`query_engine.py`)

在 `GraphStore` 之上构建的高级查询接口，面向应用层提供 6 种查询能力。

#### 查询类型

| 方法 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `find_entity` | 模糊实体搜索 | 名称关键词 | 匹配实体列表（含 `_match_score`） |
| `get_entity_relations` | 一跳关系查询 | 实体 ID + 可选关系类型过滤 | 实体信息 + 邻居列表 + 邻居数 |
| `check_relation` | 两实体路径查询 | 源 ID + 目标 ID + 最大跳数 | 是否直接关联 + 路径数 + 具体路径 |
| `search_candidates` | 关键词候选搜索 | 自然语言查询 | 候选实体列表 + 各自邻居 |
| `get_entity_context` | 实体上下文获取 | 实体 ID + 深度 | 实体 + 直接邻居信息 |
| `get_graph_summary` | 图谱概览 | 无 | 节点数、边数、类型分布、密度 |

#### 实现细节

**候选搜索（`search_candidates`）**：
- 将查询拆分为关键词（`query.lower().split()`）
- 每个关键词独立搜索，合并结果（取最高 `_match_score`）
- 按分数降序排列，返回 top-N 候选及其邻居摘要
- 适用于自然语言查询（如 "question answering" 会匹配 SQuAD、Natural Questions 等）

**路径查询（`check_relation`）**：
- 调用 `store.find_path` 获取所有 ≤ max_hops 的路径
- 判断是否存在长度为 2 的路径（直接关联）
- 返回 `min_hops`（最短路径长度）

**图谱概览（`get_graph_summary`）**：
- 计算图密度：`edges / (nodes * (nodes - 1))`
- 提供节点类型和边类型的分布统计

### 5. 查询演示 (`query_demo.py`)

- 加载图谱后执行 5 类查询测试
- 输出结构化查询报告 JSON

## 技术亮点

1. **抽象接口设计**：`GraphStore` ABC 定义 11 个标准方法，使上层代码（查询引擎、可视化）与底层存储解耦。切换 Neo4j 只需替换一行初始化代码。
2. **多级索引**：名称索引（小写映射）+ 类型索引 + 邻接表，覆盖精确搜索、模糊搜索、类型过滤、邻居查询等场景。
3. **智能名称搜索**：三级策略（精确 → 模糊 → 子串），SequenceMatcher 阈值 0.6，对中英文实体名称均有效。
4. **边去重**：同 (source, target, relation) 的边自动去重，保留置信度最高的，保证图谱质量。
5. **Neo4j 可选**：通过 try/except 处理依赖，未安装 neo4j 驱动时不影响 JSON 存储的正常使用。
6. **候选搜索**：支持自然语言查询拆词 → 模糊匹配 → 排序，面向 RAG 场景设计。
