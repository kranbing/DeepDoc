# 第三人 任务总述

## 1. 任务原文

第三人：设计结构可视化前后端处理模式，增强测试数据集（跨章节、长文档依赖），设计量化评估指标（幻觉程度、chunk 命中率），设计 RAG 与 LAD-RAG 测试集对比，分析场景优劣及检索提升率，生成对比报告一份。

## 2. 任务原文中做出来的展示内容有哪些

- 基于 QASPER 构建了 LAD-RAG 测试数据集，覆盖 20 篇论文、52 个 QA 样本和 946 个结构化 chunk。
- 测试集中保留了问题类别和难度信息：
  - `single_section`
  - `cross_section`
  - `fact`
  - `method`
  - `multi_evidence`
  - `comparison`
  - `dataset`
- 增强了跨章节和多证据问题，用于测试长文档依赖和跨章节检索能力。
- 设计了 LAD-RAG 与传统 RAG 的对比实验。
- 设计并使用了量化评估指标：
  - Chunk Recall
  - Token Recall
  - Section Recall
  - MRR
  - Hit Rate
  - Precision / Recall / F1
  - Text Overlap
- 对比分析了 RAG 与 LAD-RAG 在不同场景下的优劣：
  - 资源受限场景。
  - 单章节问题。
  - 跨章节问题。
  - 事实型问题。
  - 方法型问题。
- 输出了最终对比报告，分析 LAD-RAG 的提升率和适用边界。

## 4. 产出的任务结果的位置和分析

- `test/8-lad_rag_test/lad_vs_trad/FINAL_EXPERIMENT_REPORT.md`
  - 第三人最核心的对比报告。
  - 展示 Traditional RAG 与 LAD-RAG 在 total=10、15、20 下的效果变化。

- `test/8-lad_rag_test/lad_vs_trad/lad_rag_final_comparison_report.json`
  - 最终对比实验 JSON 结果。

- `test/8-lad_rag_test/lad_vs_trad/lad_rag_vs_traditional_report.json`
  - 初版 RAG 与 LAD-RAG 对比结果。

- `test/8-lad_rag_test/improved_eval/lad_rag_improved_eval_report.json`
  - 改进评估结果。

结果分析：

- 在 total=10 的资源受限场景下，LAD-RAG 的 Chunk Recall 相比传统 RAG 有明显提升，报告中体现为约 +10.28%。
- 在 total=15 的标准场景下，LAD-RAG 仍有一定提升，报告中体现为约 +4.11%。
- 在 total=20 的上下文较充足场景下，传统 RAG 覆盖更广，LAD-RAG 可能因结构扩展集中在少数章节而出现下降。
- 单章节问题中 LAD-RAG 更有优势，因为结构扩展可以补足同章节上下文。
- 跨章节问题中传统 RAG 更稳定，因为答案证据分散在多个章节时，过强的 section-first 扩展可能漏掉其他章节。
- 事实型问题较适合 LAD-RAG；方法型问题可能需要更宽的检索范围。


然后看最终对比脚本和报告：

```text
test/8-lad_rag_test/lad_vs_trad/lad_rag_final_comparison.py

```

第三人的重点是说明 LAD-RAG 相比传统 RAG 在什么场景下有效、提升多少、什么场景下不适合。当前报告已经按 chunk 数量、问题难度和问题类别分析了 RAG 与 LAD-RAG 的差异，也给出了后续策略选择建议。

结构可视化方面，本周主要完成数据基础和前后端处理模式设计支撑：通过 LAD chunk 中的章节、层级、位置和邻接字段，为后续图结构展示、章节定位和证据路径可视化提供数据基础。
