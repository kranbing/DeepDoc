# 第二人 任务总述

## 1. 任务原文

第二人：实现 Rerank 模块，不同策略精排召回 Top-N，在向量检索完备基础上，实现多路检索设计，设计融合策略，如加权、交集、RRF 等。记录融合策略的效果对比，rerank 策略对比，拓展 RAG 接口，形成对比报告。实现策略自主选择。

## 2. 任务原文中做出来的展示内容有哪些

- 在已有向量检索基础上，扩展了 BM25、结构分数、向量分数等多路检索信号。
- 实现了多路检索融合思路，包括：
  - 向量分数与 BM25 加权融合。
  - 结构信息加权。
  - 公式化 rerank。
  - Cross-Encoder rerank 对比。
  - RRF / 多信号融合思想。
- 实现了不同 rerank 策略下的 Top-N 召回测试。
- 对比了公式 rerank 与 Cross-Encoder rerank 的效果。
- 对种子召回数量、结构扩展方式和 rerank 后的 Top-N 召回效果做了实验。
- 输出了 rerank 策略对比报告和参数调优报告。
- 通过参数和策略实验，为后续“策略自主选择”提供依据：
  - 关键词明显时提高 BM25 或关键词信号权重。
  - 语义表达变化较大时提高向量相似度权重。
  - 需要首位证据块更准确时启用 Cross-Encoder rerank。
  - 追求运行速度时使用公式化 rerank。

## 4. 产出的任务结果的位置和分析

- `test/8-lad_rag_test/rerank/lad_rag_rerank_report.json`
  - Rerank 策略对比结果。

- `test/8-lad_rag_test/param_tuning/EXPERIMENT_REPORT.md`
  - 参数调优结果。
  - 其中 `seed8_section_first` 表现较好，Chunk Recall 达到约 0.6265。

结果分析：

- Cross-Encoder rerank 相比公式 rerank 在部分实验中能提升排序质量，尤其是首个证据块排序和 MRR。
- 多路融合的关键是平衡语义相似度、关键词匹配和结构位置，单一路径检索难以适配所有问题。
- 参数调优结果说明，seed 数量、扩展策略和 rerank 方法会影响最终召回质量，需要根据问题类型和资源限制选择不同参数。
- 策略自主选择的依据已经形成：问题类型、关键词强弱、是否需要首位证据精排、运行成本限制，都会影响采用哪一种融合和 rerank 策略。
