# 第一人 任务总述

## 1. 任务原文

第一人：设计文档结构元数据体系，补充上下文位置字段，将结构信息融入 chunk，设计结构感知策略，为结构可视化做数据基础。设计结构感知检索逻辑，设计包含标题增强、章节增强等策略检索，输出测试 demo。

## 2. 任务原文中做出来的展示内容有哪些

- 构建了 QASPER 文档的 LAD 结构化语料，形成带结构元数据的 chunk 数据。
- 在 chunk 中补充了结构和上下文位置字段，例如：
  - `chunkId`
  - `chunkKey`
  - `docId`
  - `pageNo`
  - `index`
  - `globalIndex`
  - `sectionId`
  - `sectionPathText`
  - `headingText`
  - `prevGlobalChunkId`
  - `nextGlobalChunkId`
  - `sourceChunkIds`
  - `sourceSectionIds`
  - `sourceSectionPathTexts`
- 输出了面向结构可视化和结构感知检索的数据基础：
  - `qasper_lad_corpus.json`
  - `qasper_lad_manifest.json`
  - 每篇文档独立的 `lad_chunk.json`
- 设计并测试了多种结构感知检索策略：
  - `baseline_vector`
  - `hybrid_vector_bm25`
  - `section_constrained`
  - `hierarchical_expansion`
  - `multi_level_structure`
  - `structure_aware_reranking`
- 结构策略中体现了标题增强、章节增强、同章节扩展、层级结构扩展、结构重排等思路。
- 输出了结构感知检索测试 demo 和结构检索报告。

## 3. 任务的实现文件位置与功能

- `test/8-lad_rag_test/data_prep/prepare_qasper_lad.py`
  - 将 QASPER 数据处理为 LAD-RAG 可用语料。
  - 生成带结构字段的 chunk、manifest 和文档级 `lad_chunk.json`。

- `test/8-lad_rag_test/data/qasper_lad_corpus.json`
  - 结构化后的 QASPER 文档语料。
  - 保存文档、章节、chunk、上下文位置关系等信息。

- `test/8-lad_rag_test/data/qasper_lad_manifest.json`
  - LAD chunk 的索引清单。
  - 用于快速查看所有文档与 chunk 元数据。

- `test/8-lad_rag_test/data/documents/*/lad_chunk.json`
  - 每篇论文独立的 LAD chunk 数据。
  - 可作为结构可视化和检索测试的数据输入。

- `test/8-lad_rag_test/structural/lad_rag_structural_test.py`
  - 结构感知检索测试主脚本。
  - 对比不同结构策略在 QASPER 测试集上的检索效果。

- `test/8-lad_rag_test/structural/optimized_lad_rag_strategy.py`
  - 优化后的结构感知策略实现。
  - 将结构扩展、章节优先、邻接关系等组合为更稳定的检索策略。

- `test/8-lad_rag_test/structural/test_optimized_strategy.py`
  - 优化策略对比测试脚本。
  - 验证优化策略相对基线策略的效果。

## 4. 产出的任务结果的位置和分析

- `test/8-lad_rag_test/structural/lad_rag_structural_report.md`
  - 结构感知检索测试报告。
  - 展示不同结构策略在 top-k 下的 Precision、Recall、F1、Text Overlap、MRR 等指标。

- `test/8-lad_rag_test/structural/lad_rag_structural_report.json`
  - 结构检索测试的完整 JSON 结果。

- `test/8-lad_rag_test/structural/optimized_strategy_comparison.json`
  - 优化结构策略与基线策略的对比结果。

- `test/8-lad_rag_test/structural/LAD_RAG_Structural_Retrieval_Summary.md`
  - 对结构检索实验的总结说明。

结果分析：

- `hybrid_vector_bm25` 和 `multi_level_structure` 在整体 Recall、Text Overlap 和 MRR 上表现较好，说明单纯向量检索之外加入关键词和结构信息可以提升覆盖能力。
- `section_constrained` 更强调章节内一致性，适合局部证据问题，但在跨章节问题上可能牺牲召回。
- `hierarchical_expansion` 和 `structure_aware_reranking` 为后续 LAD-RAG 的结构扩展和重排提供了策略基础。
- 结构元数据已经能够支撑后续的结构可视化、章节定位、邻接 chunk 扩展和结构感知检索。
