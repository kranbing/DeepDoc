# 第一人任务总述

## 1. 任务原文

第一人：设计 Prompt 模板，针对场景进行任务分析设计，定义不同任务下的输出范式，输出 prompt 调优报告。增强数据集，增加生成评估（正确、相关等指标），对比分析日志，探索误差案例问题来源，总结任务表现情况。

本项目中进一步细化为：在 `test` 文件夹中进行第一人的任务实现，策略是“定义任务类型 -> 设计提示词 -> 根据 QASPER 数据集，根据任务类型确定 RAG 或 LADRAG 测试提示词在对应任务下的效果 -> 根据测试结果分析提示词修改方向 -> 更新提示词 -> 重新测试”。报告要求包含定义的任务类型、每轮提示词、每轮修改点、测试结果，并且报告中展示中文提示词，脚本实际测试使用英文 Prompt。

## 2. 任务原文中做出来的展示内容有哪些

- 定义了 6 类任务类型：`local_evidence_qa`、`multi_evidence_qa`、`method_qa`、`comparison_qa`、`dataset_qa`、`fact_qa`。
- 为每类任务设计了多轮 Prompt，目前包含 Round 1、Round 2、Round 3。
- Prompt 文件中同时保留英文测试 Prompt 和中文报告展示 Prompt。
- 根据任务类型自动选择 RAG 或 LADRAG。
- 基于 QASPER 测试集生成每轮测试结果。
- 将报告拆成“相邻两轮一次调优闭环”：
  - 第 1 次：Round 1 -> Round 2
  - 第 2 次：Round 2 -> Round 3
- 报告中包含调优前测试、问题分析、调优后 Prompt、调优后测试和有效性结论。
- 修正了误判问题：区分“检索效果提升”和“Prompt 控制增强”，避免仅因 Prompt 增加结构化约束就判定任务整体有效。

## 3. 任务的实现文件位置与功能

- `test/9-first_person/run_prompt_cycle_eval.py`
  - 第一人实验主脚本。
  - 读取 QASPER LAD 数据。
  - 执行任务分类、RAG/LADRAG 策略选择、Prompt 轮次测试。
  - 计算检索指标、生成指标和 Prompt 控制指标。
  - 输出 JSON 和 Markdown 报告。

- `test/9-first_person/prompts/round_1.json`
  - 第一版 Prompt。
  - 作为初始提示词基线。

- `test/9-first_person/prompts/round_2.json`
  - 第二版 Prompt。
  - 包含相对 Round 1 的修改点。
  - 强化 JSON 输出、引用字段和证据不足处理。

- `test/9-first_person/prompts/round_3.json`
  - 第三版 Prompt。
  - 包含相对 Round 2 的修改点。
  - 强化 claim-level 证据核验和数据集类问题策略调整。

## 4. 产出的任务结果的位置和分析

- `test/9-first_person/results/prompt_cycle_eval_report_round_1_to_2.md`
  - 第 1 次 Prompt 调优报告。
  - 内容为 Round 1 测试、问题分析、修改到 Round 2、Round 2 测试和有效性分析。

- `test/9-first_person/results/prompt_cycle_eval_report_round_2_to_3.md`
  - 第 2 次 Prompt 调优报告。
  - 内容为 Round 2 测试、问题分析、修改到 Round 3、Round 3 测试和有效性分析。

- `test/9-first_person/results/prompt_cycle_eval_report.md`
  - 最新一次 Prompt 调优报告。
  - 当前指向第 2 次调优报告。

- `test/9-first_person/results/prompt_cycle_eval_report.json`
  - 完整结构化实验结果。
  - 记录所有 round 的 summary 和 case 级结果。

分析结论：

- 第一轮到第二轮主要提升了结构化输出、引用约束和证据不足处理能力。
- 第二轮到第三轮主要增强了 Prompt 控制能力，但检索证据效果并非所有任务都有实质提升，因此报告中将“Prompt 约束增强”和“任务效果提升”分开判断。
- 数据集类、局部证据类问题仍主要受检索覆盖影响，后续若继续优化，需要结合第二人的分发策略和第三人的证据链一致性结果。

## 5. 给老师的检查阐述

老师检查时可以先看 `test/9-first_person/results/prompt_cycle_eval_report.md`，这是最新一次调优报告。报告按照“调优前测试 -> 问题分析 -> 修改提示词 -> 调优后测试 -> 有效性结论”的顺序组织，能直接看到每轮 Prompt 为什么改、改了什么、指标是否改善。

如果需要看完整过程，可以继续查看 `prompt_cycle_eval_report_round_1_to_2.md` 和 `prompt_cycle_eval_report_round_2_to_3.md`。如果需要复现实验，运行：

```powershell
python test\9-first_person\run_prompt_cycle_eval.py
```

该任务的重点不在真实调用大模型，而在 Prompt 设计、任务类型定义、RAG/LADRAG 选择策略、Prompt 调优闭环和评估报告生成。当前结果已经形成可复现的第一人 Prompt 调优实验链路。
